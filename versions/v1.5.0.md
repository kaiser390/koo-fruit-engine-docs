# v1.5.0 — 2026-04-29

Live colour editing + per-banana scope hardening. Every colour
swatch and the `🎨 Color Style` palette dropdown now reach the
active banana's surface in milliseconds instead of triggering a
3-second mesh rebuild. Multi-banana scenes stay isolated through
every operator entry point.

## Highlights

- **Live colour** — colour sliders (Outer / Inner / Tip / Membrane /
  Pulp / Warm) and the Color Style palette update the active banana
  in ~30 ms via vertex-attribute foreach_set + BSDF default_value
  swap. No mesh rebuild, no transform reset, no spawning of a stray
  default banana when a pinned banana is selected.
- **ML auto-trace by default** — bundled `u2netp` model (~4.5 MB)
  replaces BiRefNet for silhouette extraction. ~80× faster (warm
  inference 0.35 s vs 29 s) and small enough to ship inside the
  extension. Subprocess-isolated so onnxruntime DLL state never
  destabilises Blender.
- **Random Generation = +1 banana** — clicking 🎲 Random Generation
  now adds a fresh randomised banana to the scene instead of
  overwriting the existing default. The previous default banana is
  auto-pinned aside before the random rolls so multi-banana variety
  packs build up with one click each.
- **Selected Banana panel — single button** — the redundant
  `Load to Panel` and `Update Snapshot` buttons are gone. Selecting
  a banana mirrors its state automatically via msgbus auto-sync;
  the lone `Apply Mesh Changes` button is reserved for shape commits
  (length / radius / curvature / n_carpels).

## Changed

- `properties/_base.py` — new `_update_color_live` callback wired
  to every FEColor swatch, the override_raw toggle, and the four
  procedural shapers (ripeness / tip_green_frac / tip_green_shift /
  saturation_scale). Falls back to the heavy regen path automatically
  when a legacy banana mesh lacks the new `fe_color_u` /
  `fe_color_outer` attributes.
- `op_stylized/randomize.py` — auto-pin existing default before the
  random rolls so the new build lands at origin without erasing the
  prior fruit. Curated random palette runs every click (no more two
  consecutive identical-coloured bananas).
- `op_stylized/random_color.py` — Color Style palette short-circuits
  to the live recolour path entirely when only colour changed
  (3.5 s → 33 ms, ~100× faster).
- `op_realistic_to_stylized.py` — converter scopes the rebuild to
  the active banana's prefix; pinned siblings stay untouched.
- `op_generate_v2.py` — captures location / rotation / scale + the
  collection memberships of the existing PeelVolume before cleanup,
  restores them on the freshly built one. Every in-place rebuild
  path now preserves diorama placement.
- `persistence/save_load.py` — `_build_payload` explicitly writes
  `banana.peel.strip{N}_amount` / `_curl` for N=4..12. The parameter
  registry only declares strip 1-3, so 5-strip bananas were losing
  strip 4-5 silently on selection swap.
- `ui/panel.py` — Selected Banana box pruned to a single
  `Apply Mesh Changes` button. Color picker rows always render
  (no longer gated on advanced + override_raw).

## Added

- `primitives/common/peel_volume/recolor.py` — vectorised vertex
  recolour + Principled BSDF Base Color updater for PeelInnerMat,
  MembraneMat, and PulpMat. Material lookup is regex-based to
  tolerate Blender's auto-appended `.NNN` suffix on cloned
  materials (Pin always triggers the suffix).
- `data/ml_models/u2netp.onnx` — bundled inference model.
- `tools/test_regressions.py` — 14 automated MCP-driven regression
  tests for the colour live path, per-banana scope, transform
  preservation, snapshot strip 4-12, and panel UI lock-in.

## Fixed

- Color slider drag on a pinned banana modified the unpinned default
  banana instead — `_regen_dispatch_op` was reading
  `view_layer.objects.active` at timer-fire time when the timer's
  context had drifted. Capture the prefix at slider-callback time
  and close over it in the debounce timer.
- Color Style dropdown spawned a brand-new default banana every
  click; now scoped to the active banana via `target_prefix`.
- Random Generation overwrote the existing default banana — now
  auto-pins first.
- Pinned banana's colours leaked into / from siblings on selection
  swap due to per-property `_update_color_live` firing during the
  random palette batch-write. `_BATCH_LOADING` flag now wraps both
  randomize_stylized and random_color batch-writes; the flag is
  guarded by try/finally so a mid-batch raise can no longer leave
  it stuck at True.
- After Pin renamed materials to `MembraneMat.001`, every recolour
  on the pinned banana silently no-op'd. Material lookup tolerates
  the `.NNN` suffix.
- Strip 4-12 amounts vanished from snapshot on a 5-strip banana
  because the parameter registry only declared strip 1-3 — sibling
  reads got `0.0` and 2 strips silently re-closed.

## Validation

14/14 automated regression tests pass against Blender 5.0.1
(~77 s). Manual viewport verification confirmed every fix above.
